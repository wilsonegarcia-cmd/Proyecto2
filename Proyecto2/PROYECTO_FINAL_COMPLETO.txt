FASE 3: ESP32 (1-2 HORAS)
═══════════════════════════════════════════════════════════════════════════════

PASO 1: Instalar ESP-IDF v5.1+
  Windows:
    • Descargar: https://github.com/espressif/esp-idf/releases
    • O usar: Installer oficial
    • Agregar a PATH
  
  Verificar instalación:
    $ idf.py --version
  
  Esperado:
    ESP-IDF v5.1.x (o superior)

PASO 2: Crear Proyecto ESP-IDF
  $ mkdir mi_proyecto_esp32
  $ cd mi_proyecto_esp32
  $ idf.py create-project-from-template system_espressif.ble_app_server

  O copiar estructura manualmente:
    $ mkdir -p esp32/main
    $ touch esp32/main/CMakeLists.txt
    $ touch esp32/main/main.c

PASO 3: Obtener Código
  El código completo está en: CODIGO_ESP32.txt
  O en archivo: c:\Users\Eduardo Garcia\Desktop\Proyecto2\esp32\
  
  Copiar el código en: esp32/main/main.c

PASO 4: Configurar
  Editar esp32/main/main.c y buscar estas líneas:
  
  Línea ~20: WiFi SSID
    #define WIFI_SSID "TU_RED_WIFI"
    
  Línea ~21: WiFi PASSWORD
    #define WIFI_PASSWORD "TU_CONTRASEÑA"
    
  Línea ~22: Servidor
    #define SERVER_IP "192.168.X.X"  // Tu IP local
    #define SERVER_PORT 5000

  Para obtener tu IP local en Windows:
    $ ipconfig
    Busca: IPv4 Address: 192.168.x.x (o 10.0.x.x)

PASO 5: Compilar
  $ idf.py build
  
  Esperado:
    ✓ Project built successfully
    
  Si hay errores de compilación:
    $ idf.py menuconfig
    • Verificar Version C++ estándar (C11+)
    • Verificar componentes habilitados

PASO 6: Subir a ESP32
  Conecta ESP32 por USB
  
  Identificar puerto COM:
    Windows: Device Manager → Ports → COM3 (o similar)
  
  Subir código:
    $ idf.py -p COM3 flash
    
  Esperado:
    ✓ Hard resetting esp32...
    ✓ Leaving... (espera 5 segundos)
    ✓ LED azul enciende (sistema listo)

PASO 7: Verificar
  $ idf.py -p COM3 monitor
  
  En consola deberías ver:
    I (XXX) BT: Bluetooth Initialized
    I (XXX) WIFI: WiFi Initialized
    I (XXX) BT_SPP: SPP listening on RFCOMM
    I (XXX) WIFI: Connected to WiFi

FASE 4: PRUEBA INTEGRADA (30 MINUTOS)
═══════════════════════════════════════════════════════════════════════════════

Checklist antes de probar:

  ☐ Terminal 1: Servidor Flask ejecutándose (puerto 5000)
  ☐ Terminal 2: idf.py monitor muestra LED azul
  ☐ Android: App instalada en dispositivo
  ☐ ESP32: Conectado por USB (para ver logs)
  ☐ Dispositivo Android: WiFi 2.4GHz conectado (misma red que servidor)

PRUEBA 1: Buscar Dispositivo
  1. Abre la app Android
  2. Haz clic en "🔍 Buscar Dispositivos"
  3. Espera 10-15 segundos
  4. Deberías ver "ESP32-Acceso" en la lista
  
  Si NO aparece:
    ✓ Verificar: Bluetooth ON en ESP32 (LED azul parpadeando)
    ✓ Verificar: Bluetooth ON en Android
    ✓ Verificar: App tiene permiso de Bluetooth
    ✓ Reinicia ESP32 y prueba de nuevo

PRUEBA 2: Conectar
  1. Selecciona "ESP32-Acceso" de la lista
  2. Haz clic en "🔗 Conectar"
  3. Espera 3-5 segundos
  4. Deberías ver "✓ Conectado" en la app
  
  Si falla:
    ✓ Verificar: Logs en Terminal 2 (idf.py monitor)
    ✓ Reintentar: A veces toma 2-3 intentos
    ✓ Esperar: 3 segundos después de seleccionar

PRUEBA 3: Token Autorizado
  1. En app Android, escribe: TOKEN123
  2. Haz clic en "✓ Verificar Acceso"
  3. Espera 2-3 segundos
  
  Esperado:
    • App muestra: GRANTED (en verde)
    • ESP32 LED: VERDE (3 segundos)
    • ESP32 Buzzer: Sonido corto (150ms)
    • Terminal 2: Log de intento

PRUEBA 4: Token No Autorizado
  1. En app Android, escribe: INVALID
  2. Haz clic en "✓ Verificar Acceso"
  3. Espera 2-3 segundos
  
  Esperado:
    • App muestra: DENIED (en rojo)
    • ESP32 LED: ROJO (3 segundos)
    • ESP32 Buzzer: Sonido largo (600ms)
    • Terminal 2: Log de intento

PRUEBA 5: Registrar Nuevo Usuario (Optional)
  En otra terminal (Terminal 3):
    $ curl -X POST http://localhost:5000/usuarios \
      -H "Content-Type: application/json" \
      -d '{"token": "NUEVO_TOKEN", "nombre": "Mi Usuario"}'
    
    Esperado:
      {"status": "agregado"}
  
  Luego intenta en Android con "NUEVO_TOKEN"
    • Debería mostrar GRANTED
