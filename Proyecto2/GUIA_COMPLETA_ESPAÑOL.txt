═══════════════════════════════════════════════════════════════════════════════

╔════════════════════════════════════════════════════════════════════════════╗
║ PASO 3: CONFIGURAR ESP32 (1-2 horas)                                     ║
╚════════════════════════════════════════════════════════════════════════════╝

GUÍA COMPLETA EN: ESP32_CONFIGURACION.txt
CÓDIGO LISTO EN: CODIGO_ESP32.txt

RESUMEN RÁPIDO:

1. Instalar ESP-IDF v5.1+
   Descargar de: https://github.com/espressif/esp-idf/releases

2. Crear estructura:
   c:\Proyecto2\esp32\
   ├── main\main.c          ← Copiar del manual o CODIGO_ESP32.txt
   ├── CMakeLists.txt       ← Crear
   └── (más archivos)

3. Configurar credenciales en main.c:
   
   #define SSID "Tu_Red_WiFi"         ← TU WiFi
   #define PASSWORD "TuContraseña"    ← TU contraseña
   #define SERVER "http://192.168.X.X:5000"  ← TU IP local

4. Configurar menuconfig:
   idf.py menuconfig
   - Habilitar Bluetooth BR/EDR
   - Guardar

5. Compilar:
   idf.py build

6. Subir firmware:
   idf.py -p COM3 flash
   (cambiar COM3 si es necesario)

7. Verificar:
   idf.py -p COM3 monitor
   Debe mostrar: "Sistema listo. Conéctate de la app Android."

RESULTADO ESPERADO:
  ✓ LED AZUL encendido (sistema listo)
  ✓ Dispositivo aparece como "ESP32-Acceso" en Bluetooth
  ✓ Logs muestran "Sistema listo"

╔════════════════════════════════════════════════════════════════════════════╗
║ PASO 4: PRUEBA COMPLETA (5 minutos)                                      ║
╚════════════════════════════════════════════════════════════════════════════╝

REQUISITOS:
  ✓ Servidor corriendo (Terminal 1)
  ✓ App Android instalada en dispositivo
  ✓ ESP32 encendido con WiFi conectado

PRUEBA:

1. En la app Android:
   - Clic "Buscar Dispositivos"
   - Debe aparecer "ESP32-Acceso"

2. Seleccionar y clic "Conectar"
   - Debe decir "✓ Conectado"

3. Ingresar token: TOKEN123
   - Clic "Solicitar Acceso"
   - Resultado: GRANTED (verde)
   - LED ESP32: VERDE
   - Buzzer: Sonido corto (150ms)

4. Ingresar token: INVALID
   - Clic "Solicitar Acceso"
   - Resultado: DENIED (rojo)
   - LED ESP32: ROJO
   - Buzzer: Sonido largo (600ms)

5. Verificar en servidor:
   curl http://localhost:5000/log
   Debe mostrar los 2 intentos registrados

RESULTADO FINAL:
  ✅ ¡SISTEMA COMPLETAMENTE FUNCIONAL!

🛠️ SOLUCIONAR PROBLEMAS

SERVIDOR:
  P: "Port already in use"
  R: Editar app.py > cambiar puerto 5000 a 5001

ANDROID:
  P: "Gradle build failed"
  R: Build > Clean > Rebuild

  P: "Cannot resolve symbol R"
  R: File > Invalidate Caches / Restart

ESP32:
  P: "Serial port not found"
  R: Verificar que USB está conectado
     Device Manager > Ports para ver qué puerto es

  P: "WiFi no conecta"
  R: Cambiar a red 2.4GHz (no 5GHz)
     Verificar SSID y PASSWORD

📱 FLUJO DE UN ACCESO

1. Usuario abre app Android
2. App busca dispositivos Bluetooth
3. Encuentra "ESP32-Acceso"
4. Usuario conecta
5. Se establece conexión SPP
6. Usuario ingresa TOKEN123
7. App envía token al ESP32
8. ESP32 recibe token
9. ESP32 consulta servidor HTTP: /check?token=TOKEN123
10. Servidor consulta BD: ¿TOKEN123 está autorizado?
11. Servidor responde: {"autorizado": true}
12. ESP32 enciende LED VERDE
13. ESP32 emite buzzer (150ms)
14. ESP32 envía "GRANTED" a la app
15. App muestra: GRANTED (texto verde)
16. Servidor registra: token=TOKEN123, resultado=GRANTED, timestamp
